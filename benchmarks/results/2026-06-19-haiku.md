## ultravalidate pass (n=20 + cross-model + intensity + transfer)

**Verdict: `supported` (significance-tested, single-shot).** No longer "directional": at n=20 the
reduction is statistically significant on every task and reproduced on a second Claude model.

**Weakest defensible restatement.** For single-shot generation, flint cuts output at equal measured
correctness, and the effect is real, not noise: on `claude-haiku-4-5` (n=20) prose tokens fell ~33%
and code LOC ~64% (pooled normalized reduction −42%, 95% CI [−53%, −37%]), with Mann-Whitney
p < 1e-6 on six of seven tasks (slugify p = 0.011) and every gate 20/20. It reproduces and
strengthens on `claude-opus-4-8` (n=10, pooled −62%, every task significant, all gates 10/10). It
does NOT transfer to small local models: on qwen2.5-coder 3b/1.5b (Ollama, n=10) the pooled effect
is −31% / −0% but NOT significant, with prose answers failing the fidelity floor (the 1.5b rambles
+65% under the skill). flint requires an instruction-follower.

**Checks:**
1. *Reconcile from source* PASS. Per-task medians, CIs, and p-values are computed by `stats.py`
   (hand-rolled Mann-Whitney U with tie correction + seeded bootstrap) directly from the raw
   per-record `.scored.json`; the chart numbers are generated by `charts.mjs` from the same files.
2. *Fairness* PASS. Same model, prompt, base system prompt, no tools, n across arms; only the
   system-prompt text differs. flint carries a larger INPUT prompt (the SKILL), not in the metric.
3. *Power* RESOLVED for Claude. n=20 (haiku) / n=10 (opus); significant on every task; CIs exclude
   zero. The small-model arms are n=10 and explicitly NOT significant; reported as such.
4. *Confound* DISCLOSED. The operator's `~/.claude/CLAUDE.md` loads into every Claude arm (harness
   blocked relocating it); constant across arms, baseline still behaved verbosely, so the direction
   is conservative. Small-model "reductions" are partly degenerate (fidelity-failed answers), so
   they are marked not-significant and not counted as wins.
5. *Falsifier* RAN. "flint does not reduce output at equal correctness, significantly, on a capable
   model." 420 + 210 calls, correctness-gated, MWU-tested; flint reduced output on all 7 tasks at
   p < 0.05. Did not hold for Claude. It DID hold for small local models (honest negative).

**Intensity dose-response (haiku, n=10):** total output tokens baseline 2420 -> lite 1449 -> full
1373 -> ultra 1338 -> feral 1085, monotonic. lite-to-ultra hold the gate at 100% (free, down to
-45%); the experimental feral level reaches -55% but the gate drops to 77% (prose fidelity, mostly
conn-pool). Full write-up: results/2026-06-19-feral-frontier.md.

**What this still does NOT show:** the build-right (eng-audit) and claim-only-what's-proven
(ultravalidate) edges (behavioral, not size-measurable); anything multi-turn / agentic on a real
repo (single-shot understates the ladder + principles); models beyond those listed. Prior runs
preserved: `results-2026-06-19-v1.json`, `results-2026-06-19.json` (n=5).
